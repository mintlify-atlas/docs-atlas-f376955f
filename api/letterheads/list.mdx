---
title: "List letterheads"
description: "Retrieve letterheads (hojas membretadas) by area ID or paginate all letterheads with optional filtering."
---

## Endpoints

| Method | Path | Description |
|---|---|---|
| `GET` | `/membretadas/area/:areaId` | Get all letterheads for a specific municipal area. |
| `GET` | `/membretadas` | List all letterheads with optional filtering and pagination. |
| `GET` | `/membretadas/:id` | Fetch a single letterhead by ID. |

All endpoints require a valid JWT token in the `Authorization` header.

<Warning>
  This API is hosted on Render.com's free tier (Auth server: `https://login-pwa-atoto.onrender.com`). If the service has been idle, the first request may take up to **30 seconds** while the server cold-starts.
</Warning>

## Get letterheads by area

```
GET https://login-pwa-atoto.onrender.com/membretadas/area/{areaId}
```

Returns all letterheads that belong to a given municipal area, without pagination. Useful for populating a letterhead picker when creating or editing a machote.

### Path parameters

<ParamField path="areaId" type="string" required>
  The identifier of the municipal area. Corresponds to the `areaKey` values used in the machotes API: `RH`, `SEC`, `TES`, `OP`, `DS`, `SP`, `CAT`, `SERV`.
</ParamField>

### Example

```bash
curl "https://login-pwa-atoto.onrender.com/membretadas/area/RH" \
  -H "Authorization: Bearer <token>"
```

### Response

```json
        "publicId": "membretadas/rh_oficial_2025",
        "formato": "pdf",
        "bytes": 245760
```

<ResponseField name="areaId" type="string">
  The area identifier echoed from the request.
</ResponseField>

<ResponseField name="total" type="number">
  Total number of letterheads in this area.
</ResponseField>

<ResponseField name="hojas" type="HojaMembretadaAPI[]">
  Array of letterhead objects.

  <Expandable title="HojaMembretadaAPI fields">
    <ResponseField name="id" type="string">
      Unique letterhead ID. Use this in GET/DELETE requests.
    </ResponseField>

    <ResponseField name="areaId" type="string">
      The municipal area this letterhead belongs to.
    </ResponseField>

    <ResponseField name="nombre" type="string">
      Display name of the letterhead.
    </ResponseField>

    <ResponseField name="descripcion" type="string">
      Optional description of the letterhead's intended use.
    </ResponseField>

    <ResponseField name="archivo" type="object">
      File metadata stored in Cloudinary.

      <Expandable title="archivo fields">
        <ResponseField name="url" type="string">
          Original file URL. May point to a PDF — use this for download links only.
        </ResponseField>

        <ResponseField name="previewUrl" type="string">
          JPEG image URL generated by Cloudinary. Always safe to use in an `<img>` tag, regardless of the original file format.
        </ResponseField>

        <ResponseField name="publicId" type="string">
          Cloudinary public ID. Used internally for deletion and transformation.
        </ResponseField>

        <ResponseField name="formato" type="string">
          Original file format: `png`, `jpg`, `webp`, or `pdf`.
        </ResponseField>

        <ResponseField name="bytes" type="number">
          File size in bytes.
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="activa" type="boolean">
      Whether the letterhead is currently active and available for use in machotes.
    </ResponseField>

    <ResponseField name="subidaPor" type="object">
      Identity of the user who uploaded the letterhead.

      <Expandable title="subidaPor fields">
        <ResponseField name="nombre" type="string">First name of the uploader.</ResponseField>
        <ResponseField name="apellidos" type="string">Last name(s) of the uploader.</ResponseField>
        <ResponseField name="puesto" type="string">Job title of the uploader.</ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="creadoEn" type="string">
      ISO 8601 timestamp of when the letterhead was uploaded.
    </ResponseField>

    <ResponseField name="actualizadoEn" type="string">
      ISO 8601 timestamp of the last update.
    </ResponseField>
  </Expandable>
</ResponseField>

<Tip>
  Always use `previewUrl` in image tags — not `url`. The original `url` may be a PDF, which cannot be rendered in an `<img>` element. Cloudinary automatically generates a JPEG preview for PDF uploads and stores it in `previewUrl`.
</Tip>

## List all letterheads (paginated)

```
GET https://login-pwa-atoto.onrender.com/membretadas
```

Returns a paginated list of all letterheads. Filter by area using the optional `areaId` query parameter.

### Query parameters

<ParamField query="areaId" type="string">
  Filter results to a specific area. Omit to return letterheads from all areas.
</ParamField>

<ParamField query="page" type="number" default="1">
  Page number to retrieve. Starts at `1`.
</ParamField>

<ParamField query="limit" type="number" default="20">
  Number of results per page.
</ParamField>

### Examples

<CodeGroup>

```bash All letterheads, page 1
curl "https://login-pwa-atoto.onrender.com/membretadas?page=1&limit=20" \
  -H "Authorization: Bearer <token>"
```

```bash Filter by area
curl "https://login-pwa-atoto.onrender.com/membretadas?areaId=TES&page=1&limit=10" \
  -H "Authorization: Bearer <token>"
```

</CodeGroup>

### Response

```json

    {
      "id": "6651d4e5f6a7b8c9d0e1f2a3",
      "areaId": "TES",

      "activa": true,
      "subidaPor": {
        "nombre": "Lucía",
```

<ResponseField name="hojas" type="HojaMembretadaAPI[]">
  Array of letterhead objects for the current page. Same shape as the `hojas` array in the area endpoint above.
</ResponseField>

<ResponseField name="paginacion" type="object">
  Pagination metadata.

  <Expandable title="paginacion fields">

      Current page number.
    </ResponseField>

    <ResponseField name="paginas" type="number">
      Total number of pages.
    </ResponseField>

    <ResponseField name="limite" type="number">
      Number of results per page as requested.
    </ResponseField>
  </Expandable>
</ResponseField>

## Error responses

| HTTP status | Cause |
|---|---|
| `401` | Missing or invalid `Authorization` header. |
| `404` | No letterhead found with the given `id` or `areaId`. |
| `500` | Internal server error. Retry after a few seconds if the service just cold-started. |
